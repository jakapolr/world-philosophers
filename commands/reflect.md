The user wants the **growth mirror**.

1. Load the **growth-mirror** skill and follow it.
2. If the argument begins with `log` (or the user wants to record a session), append a
   journal entry to `journal/` using `templates/growth-entry-template.md`.
3. Otherwise, read the journal entries in `journal/` and reflect the **arc of change**
   back to the user — grounded in specific dated entries, honest, never flattering,
   never predictive. Close with a reflective question, per `PRINCIPLES.md`.
4. For the plugin owner, mirror new entries to the Obsidian vault when the MCP is
   available. For open-source users, keep the journal local.
